Getting started with Git and Github

In this page, we will be introducing to you about Git and Github which is two very important application to IT and every programmer. This program is similar to common tools such as Google Drive but it is especially built for people to work on a project or program together. Git and Github also work perfectly for people wanting to work on a coding program with helping application such as repository, pull, push and it was also supported by many coding editor for people to do their project. That is the reason why for every programmer or people who want to become a programmer, learning about these application is a must.

Why programmer choose Git and Github

As I mentioned above, Git and Github was perfect for hosting projects and people to work together. However, how is it better than normal application like Google Drive aside from the fact you can upload it from code editor such as Visual Studio Code? The reason is because it has something called repository and branching which will be explained later on. It was made just for hosting project and user-friendly for people who are working or looking for a project.

Git command

In this section, we will be learning about some of the most basic Git and github. Firstly, we will be learning the very first command that you need to do when you are working with git and github is git clone. After that, we can look at the other basic command for every person to work on a project.

Git add - commit - push combination

This time, we will be learning about the combination add-commit-push. This is an important combination for you to push all the file that you worked on in your local machine to the online repository. The reason why they dissect the ability to push your work online to 3 different parts is so that you can debug and fix your problem much easier. Each command in this combination has a different function that eventually, lead to you pushing your work online.
An important thing to notice: You have to be inside the directory so make sure that you are in the folder that you clone from the git clone command. Only by being inside the folder that you clone from github will give you the ability to use the combination.

Git pull command

the next command we will look at is the git pull command. This command is the opposite to the combination. If we use the combination, we can push the information from our local machine to the Github then the git pull command is for us to pull the all the changes that was made by the other to our local machine. However, unlike the combination, since the other have already fix all the bug when they use the combination when they push the changes to github, we don't need to have to go through many combination but only need to use the git pull command.
An important thing to notice: git pull also require you to be inside the same folder that you clone from github to be able to pull the changes in.
